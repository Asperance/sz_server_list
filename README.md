# Server Directory — GitHub Pages

Статическая страница со списком игровых адресов. Данные обновляются GitHub Actions каждые 15 минут.

## Что изменено в версии 2

- приоритет регионов: Россия → Европа → Америка → Юго-Восточная Азия → Северо-Восточная Азия;
- ASN, сетевой оператор, город и страна добавляются во время сборки;
- адреса источников и время сетевого отклика не публикуются в интерфейсе;
- сетевые данные кэшируются на 30 дней;
- повторные API-запросы выполняются только для новых или устаревших IP;
- workflow использует Node.js 24.

## Обновление существующего репозитория

Замените следующие файлы и каталоги:

- `.github/workflows/pages.yml`
- `scripts/fetch-servers.mjs`
- `public/index.html`
- `public/app.js`
- `public/styles.css`
- `public/data/servers.json`

Добавьте новый файл:

- `public/data/ip-cache.json`

После commit откройте **Actions**, выберите workflow **Update server list and deploy Pages** и запустите **Run workflow**.

## Автоматическое обновление

Workflow запускается на 7-й, 22-й, 37-й и 52-й минуте каждого часа, а также вручную.

## Сетевые данные

Во время первой сборки Action запрашивает сведения для всех уникальных IP. Результаты сохраняются в `public/data/ip-cache.json`.

Последующие сборки используют сохранённый кэш. Повторная проверка выполняется:

- для нового IP;
- если существующая запись старше 30 дней.

Если сервис геолокации временно недоступен, список серверов всё равно публикуется. Для ранее известных IP сохраняются старые сведения.

Город по IP является приблизительным и может означать точку присутствия сети, а не физический адрес конкретной машины.

## Поведение при сбоях

- если недоступен один регион, остальные регионы публикуются;
- проблемный регион отображается как временно недоступный;
- если недоступны все регионы, новый deployment отменяется;
- предыдущая рабочая версия сайта остаётся опубликованной.


## Адаптивный интерфейс версии 3

Добавлены отдельные режимы для:

- 4K и больших ультрашироких экранов;
- 2K / 2560×1440;
- Full HD / 1920×1080;
- ноутбуков 1366×768 и 1440×900;
- планшетов в альбомной и портретной ориентации;
- телефонов;
- маленьких телефонов;
- телефонов в альбомной ориентации.

На desktop используется полноценная таблица. На планшетах и телефонах строки автоматически превращаются в карточки. Также добавлены safe-area отступы, режим reduced motion и стили печати.
